# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| main    | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please follow these steps:

1. **DO NOT** open a public GitHub Issue
2. Email us at **security@ucoingroup.example** with details
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if available)

## Response Timeline

- **Initial Response**: Within 48 hours
- **Status Update**: Within 7 days
- **Resolution**: Depends on severity (Critical: 7 days, High: 14 days, Medium: 30 days)

## Disclosure Policy

We follow responsible disclosure principles:
- We will acknowledge your report promptly
- We will keep you informed of our progress
- We will credit you in our security advisories (if desired)
- We ask that you give us reasonable time to fix the issue before public disclosure

## Security Best Practices

When integrating EACO into your projects:
- Never expose private keys or seed phrases
- Use environment variables for sensitive configuration
- Validate all user inputs
- Use HTTPS for all API calls
- Keep dependencies up to date

Thank you for helping keep EACO and our community safe!
